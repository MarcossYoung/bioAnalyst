# Nullifier v7.5 

Delta on v7.1. Two additions that unblock the tool's core use case:

1. **R via rpy2 for branch-model dN/dS (ω)** — the tool can finally compute evolutionary rates, which is the entire point of testing co-evolution hypotheses.
2. **Batch Ensembl fetching** — moved from v8 to v7.5 because 60-minute first runs on expanded gene sets are unusable.

Budget limits and closed-loop discovery stay in v8.

---

## 1. Backend changes

### R integration via rpy2
New module `tools/r_bridge.py` — Python-to-R interface via rpy2.

R packages required:
- `ape` — phylogenetic tree handling, Newick I/O
- `phangorn` — tree manipulation, model fitting
- `seqinr` — sequence reading and codon-aware alignment handling
- `caper` — PGLS (deferred to v8/v9 but installed now)

rpy2 initializes R once at pipeline startup. If R is not installed or packages are missing, fail fast at health check with a clear error listing what's needed.

Config addition to `~/.nullifier/config.toml`:
```toml
[r]
enabled = true
r_home = ""  # auto-detect if empty; set manually if R is in a non-standard location
```

### PAML codeml integration
New module `tools/paml.py` — calls codeml via R's system interface or direct subprocess.

**Inputs per gene:**
- Codon-aware alignment (fetched from Ensembl Compara's pre-computed alignments via `/alignment/region` or `/genetree` endpoints; PRANK fallback for gaps)
- Newick species tree with foreground branches labeled (fetched from Ensembl gene tree, pruned to species with alignment data)
- codeml control file generated from template

**Two models run per gene:**
- Null model (model=0): single ω across all branches
- Branch model (model=2): separate ω for foreground vs background branches
- Likelihood ratio test between models → p-value for acceleration on foreground branch

**Outputs per gene (structured `PamlResult`):**
- `omega_foreground` — dN/dS on the foreground branch
- `omega_background` — dN/dS on background branches
- `lrt_statistic` — 2×(lnL_alt - lnL_null)
- `lrt_pvalue` — from chi-squared distribution (df=1)
- `alignment_length` — codons used
- `species_count` — species in tree
- `foreground_label` — which lineage was designated foreground
- `provenance` — PAML version, control file hash, alignment source, tree source

**Caching:** keyed by `(gene_id, foreground_lineage, alignment_hash)`. TTL 90 days.

**Timeout:** 5 minutes wall clock per gene. If exceeded, return structured error — don't block the pipeline.

**Foreground lineage selection:** the Methodologist proposes foreground branches based on hypothesis context (e.g., "primates" for a primate-complexity hypothesis). The confirmation gate lets the user override before PAML runs. Default foreground options available in config: primates, rodents, cetaceans, chiroptera.

### Compute layer — dN/dS-aware tests
With real ω values now available, the Compute layer gains:
- **Cross-set ω comparison:** Kruskal-Wallis on foreground ω across synaptic vs BBB vs control gene sets
- **Pairwise Mann-Whitney post-hoc** on ω distributions (with Bonferroni/BH correction)
- **Spearman correlation** of ω between paired synaptic-BBB gene sets (when pairing criteria exist)
- **Acceleration ratio:** per-gene `omega_foreground / omega_background` — genes with ratio > 1 are accelerated on the foreground branch
- **Set-level acceleration comparison:** are BBB genes more accelerated than synaptic genes on the primate branch? (Mann-Whitney on acceleration ratios)

All tests return the full `TestResult` schema (statistic, p-value, effect size, CI, correction method) — v7.1 contract enforced.

### Batch Ensembl fetching (moved from v8)
Replace per-gene sequential REST calls with batch POST endpoints:
- `/lookup/id` — up to 1,000 IDs per POST
- `/homology/id` — multiple IDs per POST

Reduces ~18,000 API calls for 3,644-gene expanded set to ~200 batched requests. First-run time drops from ~60 minutes to ~5-10 minutes.

Also add:
- `ensembl.batch_progress` events via WebSocket → UI progress bar
- Configurable `max_genes_for_full_analysis` (default 500) in config.toml — stratified subsampling when exceeded

### Tree visualization (new)
New frontend component `PhylogenyView`. Renders gene family tree with:
- Branch-model ω overlaid as branch color (blue = constrained, red = accelerated)
- Foreground lineage highlighted
- Duplication nodes annotated
- Branch ω and LRT p-value on hover

Minimal D3 tree renderer — no heavy phylotree.js dependency.

### New events
- `paml.gene_started` / `paml.gene_complete` (with gene name and ω values)
- `paml.gene_timeout` (with gene name)
- `ensembl.batch_progress` (N of M genes fetched)
- `r.initialized` / `r.error` (health check result at startup)

## 3. Frontend changes

The gene set panels that previously showed `dN/dS: n/a` now show:
- Mean foreground ω per set
- Mean acceleration ratio per set
- Individual gene ω values in a sortable table

Renders when PAML results are available. One tree per gene (expandable from gene table rows). Shows branch ω as color overlay with hover details.

Visible during large batch fetches. Shows "Fetching gene data: 342 of 3,644."

### Limitations section updated
Remove "dN/dS ratios unavailable due to Ensembl Compara data limitations." Replace with actual PAML limitations where applicable: "Branch-model ω computed via PAML codeml. Genes without sufficient alignment depth excluded."

## 4. Installation changes

v7.5 adds two new system dependencies. Document clearly in README:

**R (4.0+):**
- Mac: `brew install r`
- Ubuntu: `sudo apt install r-base`
- Windows: download from r-project.org

**R packages (one-time):**
```r
install.packages(c("ape", "phangorn", "seqinr", "caper"))
```

**PAML (codeml binary):**
- Mac: `brew install paml` or compile from source
- Ubuntu: `sudo apt install paml`
- Windows: download from abacus.gene.ucl.ac.uk

**Python:**
```
pip install rpy2
```

**Health check at startup:** `nullifier serve` verifies R is reachable, required packages are installed, and codeml binary is on PATH. Fails fast with specific missing-dependency messages if not.

## 6. Open questions

1. **Alignment source.** Ensembl Compara provides pre-computed CDS alignments for many gene families. Are these sufficient for PAML, or do we need to generate fresh alignments with PRANK? Recommend: try Ensembl first, PRANK fallback for genes where Ensembl alignment is missing or too short.

2. **Species tree source.** PAML needs a species tree. Options: (a) Ensembl gene tree pruned to species with data, (b) a canonical mammalian species tree shipped in the repo (e.g., from TimeTree). Recommend: (a) for per-gene trees, (b) as a fallback for genes without Ensembl gene trees.

3. **Minimum alignment depth.** How many species must be present in the alignment for the branch model to be meaningful? Recommend: minimum 8 species including at least 2 foreground and 4 background. Genes below this threshold get `skipped` with structured reason.
