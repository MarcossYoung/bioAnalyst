# Nullifier v6 — Concise Plan
 v6 hardens the scientific backbone:

1. **Provenance everywhere** — every claim, classification, critique carries a structured `provenance` record.
2. **Computations leave the LLM** — Analyst splits into Methodologist (LLM) → Compute layer (deterministic Python) → Interpreter (LLM).
3. **Gene-set expansion** — auto-expand from canonical sources (SynGO + hardcoded BBB sets) with matched controls.
4. **Two-verdict structure** — separate `hypothesis_verdict` and `analysis_verdict` when both apply.
5. **Decomposed scoring** — every score has explicit components + calibration disclaimer.
6. **Robustness check** — leave-one-out perturbation on starter gene sets.
7. **Tool always runs its own analysis** — user's submitted analysis is one input among many, not a substitute.


---

## 1. Backend changes

### Methodologist (new agent, Claude)
Replaces v5's Analyst-first-half. Decides which statistical tests are appropriate given the hypothesis, available data, and gene sets. Outputs a structured analysis plan. No computation here. Tests it can request come from a hardcoded library:
- Kruskal-Wallis + Mann-Whitney post-hoc
- Spearman / Pearson correlation
- Fisher's exact / chi-square
- Bonferroni and Benjamini-Hochberg correction
- Bootstrap confidence intervals
- Permutation tests
- Effect sizes (Cliff's delta, Cohen's d)

If the Methodologist requests something outside this library, the response surface is "test not available; here's the closest alternative."

### Compute layer (new, deterministic Python)
Executes the Methodologist's plan using scipy + statsmodels. **No LLM.** Returns typed, structured results with effect sizes, confidence intervals, and a complete record of what was computed.

Also handles the **reproducibility check** absorbed from v5 — verifying user-reported statistics against retrievable values is just another deterministic computation.

Also handles the **robustness check** — leave-one-out perturbation on the starter gene set. Re-runs the primary tests with each gene dropped; reports whether the verdict is stable or fragile.

### Interpreter (Claude, renamed Analyst)
Reads deterministic Compute results + genomic data. Produces interpretation: patterns observed, outlier genes, limitations. Cannot invent numbers — inputs are typed and validated.

### Gene-set expansion module (new)
`tools/gene_sets.py` exposes canonical sets:
- `synaptic.*` — fetched from SynGO API, cached locally with release version stamped, refreshed weekly
- `bbb.endothelial.*`, `bbb.astrocyte_support`, `bbb.pericyte` — hardcoded from published BBB transcriptome references, version-tracked in the repo
- `controls.*` — hardcoded brain-expressed gene sets matched on expression level, gene length, GC content, constraint, tissue specificity

The Methodologist can request `expand starter list against synaptic.* sets, match controls on expression+length`. A new local-Gemma classifier (see below) filters canonical sets for hypothesis relevance. every run, regardless of what the user attached, the Methodologist auto-expands starter entities to the largest defensible canonical set. The tool always does its own analysis on the expanded set.

### Provenance objects (new, everywhere)
Every claim, classification, critique, and score gets a structured `provenance` block:
```
{
  "source": "librarian|methodologist|compute|interpreter|skeptic|ensembl|synGO|...",
  "triggered_by": [...],
  "evidence_refs": [...],
  "method": "...",
  "confidence": 0.0-1.0,
  "pipeline_version": "v6.0.0",
  "timestamp": "...",
  "input_hash": "..."
}
```

### Two-verdict structure (Skeptic)
- `hypothesis_verdict`: based on literature + tool's own analysis. Always produced.
- `analysis_verdict`: based on critique of user's submitted analysis. Produced only when completed_analysis was attached.

Pure hypothesis runs render only `hypothesis_verdict`. Hypothesis-plus-analysis runs render both.

### Decomposed scoring
Every score is structured as:
```
{
  "value": 7,
  "components": [
    {"name": "literature_consensus", "value": 8, "evidence": "..."},
    {"name": "statistical_robustness", "value": 5, "evidence": "..."}
  ],
  "aggregation_method": "weighted_mean",
  "calibration_note": "Heuristic score, not a calibrated probability"
}
```
The calibration note renders in the UI, not buried.

### New events
- `methodologist.plan_complete`
- `compute.start` / `compute.test_complete` / `compute.all_complete`
- `compute.robustness_start` / `compute.robustness_complete`
- `gene_sets.expanded` (with source + version)
- `interpreter.start` / `interpreter.complete`

## 2. LLM routing (hybrid)

| Task | Backend | Volume per run |
|---|---|---|
| Formalizer S1 + S2 | Claude | 2 calls |
| Query Expander | Claude | 3-5 calls |
| Librarian per-paper | Gemma local | 30-50 calls |
| Librarian synthesizer | Claude | 3-5 calls |
| Methodologist | Claude | 1 call |
| Compute layer | (no LLM) | n/a |
| Interpreter | Claude | 1-2 calls |
| Skeptic | Claude | 1 call |
| **Gene-set expansion classifier** (new) | **Gemma local** | 100-500 calls |
| **Provenance enrichment** (new) | **Gemma local** | scales with all agents |
| **Robustness per-perturbation reading** (new) | **Gemma local** | 8-20 calls |
| Robustness summary | Claude | 1 call |

Three new high-volume tasks routed local offsets the cost increase from running the tool's own analysis every time. Net per-run cost roughly equal to v5 despite substantially more work.

## 3. Frontend changes

### Verdict panel — split
Two verdict cards side-by-side when both apply. Solo when only hypothesis_verdict applies. Verdict cards include the decomposed scoring breakdown (expandable) with calibration note visible.

### Compute results section (new)
Renders deterministic statistical output: tables of test results with effect sizes, confidence intervals, p-values, and correction methods. Visually distinct from LLM-generated interpretation panels — different background, "computed" tag.

### Gene-set expansion panel (new)
Shows what canonical sets were used, how many genes were included, what SynGO release version was active, control-matching parameters.

### Provenance — accessible but not loud
Every assertion in the report gets a small "provenance" affordance (info icon). Click expands the structured record. Default-collapsed; available when the user asks "why does the tool think this?"

### Robustness panel (new)
"Verdict stability: stable" / "Verdict stability: fragile (driven by SYP)". Small visualization of which genes most influence the conclusion.

### Everything else unchanged
v4/v5 components stay. Confirmation modal, flag UI, history, tool call cards — all same.

## 7. Success criteria

**Tool's own analysis works**
- Pure hypothesis input expands starter genes via SynGO + hardcoded BBB sets
- Compute layer runs the test family selected by the Methodologist with proper multiple-testing correction
- Interpreter produces honest interpretation citing the deterministic numbers
- Robustness check returns a stability rating

**Two-verdict structure works**
- Pure hypothesis run → only `hypothesis_verdict` rendered
- Hypothesis + analysis run → both rendered; `analysis_verdict` critiques user's submitted work

**Provenance works**
- Every assertion in the report has a clickable provenance record
- Father can answer "why does the tool think this?" without asking the developer

**Cost stays roughly flat from v5** despite the tool doing substantially more work.
