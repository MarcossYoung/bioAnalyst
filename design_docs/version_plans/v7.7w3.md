# Nullifier v7.7 -- W3 (Tier-1 mirrortree-lite)

## Context

W2 (see **v7.7w.md**) makes the Methodologist return `untestable` for a
`cross_lineage_rate_correlation` claim because no available test can evaluate it. W3 builds
that test, so `untestable` becomes a real STRONG / WEAK / FALSIFIED verdict on the
rate-correlation leg.

**Why the current path cannot be patched in place:** `tools/genomic_data.py`
`per_gene_metrics()` collapses each gene to a single scalar dN/dS (`mean(dnds_vals)`), mixing
paralogs and leaving saturation unfiltered. Co-evolution lives in the *per-lineage* dimension;
averaging over lineages destroys the signal before any test runs. W3 is a **parallel** genomic
path with its own typed object and its own test family -- it does not replace the scalar
set-difference path.

**Prerequisite:** W3 is gated behind W2's capability map and must land after W1+W2.

### Decisions (locked)
- **Rate-vector source = the R seqinr::kaks path we already built.** Ensembl Compara homology
  `dn`/`ds` fields come back empty (this is the v7.6R finding), so per-species pairwise dN/dS
  must come from `tools/r_bridge.pairwise_dnds(sequences)`, which runs `seqinr::kaks` on the
  Compara gene-tree codon alignment (`ensembl.fetch_gene_tree_aligned`) and returns
  `{species_lower: dnds}` per gene -- **that mapping is exactly a per-lineage rate vector.**
  Reuse it; do not re-read the empty Compara `dnds`.
- **Background set = static checked-in file** (`backend/nullifier/data/random_background_300.txt`)
  of randomly chosen protein-coding gene symbols. Deterministic across runs, matches the
  pure-compute principle, trivial to test.
- **Species panel = a new fixed 20-40 mammal panel** (broader than `paml.py`'s 17). Tier-1
  correlates per-lineage rate vectors and residualizes against background -- it does **not**
  need a tree topology. A TimeTree newick is deferred to Tier-2 ERC.

## Components

### 1. Fixed species panel
- New `backend/nullifier/data/mammal_panel.txt` (or a constant in a small `tools/panels.py`):
  20-40 well-placed mammals as Ensembl species names (lowercase, e.g. `homo_sapiens`,
  `pan_troglodytes`, `mus_musculus`, `bos_taurus`, `canis_lupus_familiaris`, ...), spanning
  primates, rodents, carnivores, ungulates, cetaceans, chiroptera.
- Every gene's rate vector is defined over the **same** panel order; missing species -> `None`.
- **Casing:** ortholog `target_species` and `pairwise_dnds` keys are lowercase; the panel must
  be lowercase. Normalize on every join.

### 2. 1:1 orthologs only
- Filter to `ortholog_type == "one2one"` and restrict to panel species. **Exclude paralogs**
  (current data has up to ~64 dups on one gene, which contaminates any correlation). This also
  keeps the two compared sets effectively disjoint, addressing the cross-set Jaccard 0.765
  self-correlation concern.

### 3. Per-gene rate vector across the panel
- New function (`tools/genomic_data.py`, e.g. `per_gene_rate_vectors(gene_data, panel)`) that,
  per gene, assembles a panel-aligned dN/dS vector. Source values from `r_bridge.pairwise_dnds`
  (per-species), filtered to one2one panel species; do **not** collapse.
- Attach as a parallel typed object on the compute `data` dict, e.g.
  `data["rate_vectors"] = {"panel": [...], "gene_index": [...], "rates": [[...], ...]}` --
  separate from `groups`/`variables` so the existing scalar path is untouched.

### 4. Background normalization (critical, easily skipped)
- Estimate the per-lineage genome-wide mean from the **large random background set** (the static
  300-gene file), then correlate **residuals**, not raw rates. Raw rates are dominated by
  lineage effects (generation time, etc.) and produce spurious positive co-evolution -- exactly
  the phylogenetic confounder the input itself listed. The 16-gene `control.housekeeping_brain`
  set is far too small for this; the background set replaces it for normalization.

### 5. Pair-level test with the right null
- Unit = gene pair. Compare **cross-set** rate covariation (Set A x Set B) against
  **within-set** and **set-vs-background** baselines.
- New pure functions in `tools/compute.py`: residualization against background, pairwise
  covariation, cross-vs-within-vs-background comparison, and a **permutation null** on the mean
  cross-set value. Reuse the existing `permutation_test` pattern and the `_adjust` BH correction
  already in `compute.py`. Keep them deterministic (same data -> same result; seed the RNG).
- Register `mirrortree_lite` in `TEST_LIBRARY` with a new `kind` (e.g. `"rate_vectors"`),
  `constructs=["cross_lineage_rate_correlation"]`, and a `_run_one` dispatch branch that reads
  `data["rate_vectors"]`. Return the standard `_test_result(...)` shape (all
  `TEST_RESULT_FIELDS`), with `statistic` = observed cross-set covariation and `p_value` from
  the permutation null. This yields a clean `FALSIFIED` when cross-set is indistinguishable from
  background -- an outcome the current setup cannot produce.

### 6. Wire into the W2 gate
- `agents/methodologist.py` capability map gains
  `cross_lineage_rate_correlation -> mirrortree_lite`, flipping the W2 `untestable` into a
  runnable plan. Update the methodologist prompt/`data_shape` doc so it knows the rate-vector
  inputs exist.

### 7. Saturation filtering of rate vectors
- Apply the W2 saturation filter to the per-species rates before testing: drop branches with
  dN/dS pinned near 1.0 so saturation does not masquerade as covariation.

## Risks
- **Compara coverage of the panel** -- not every panel species has a one2one ortholog or an
  alignment for every gene; vectors will be sparse. Require a minimum number of shared species
  per pair before testing; report coverage.
- **Rate-vector cost** -- `pairwise_dnds` runs an Rscript per gene; a panel x hundreds of
  background genes is a one-time cost. The rdnds + Ensembl caches make reruns cheap; cap gene
  counts via the existing `_limit_targets`.
- **Casing/species mismatch** -- lowercase-normalize panel, ortholog `target_species`, and
  `pairwise_dnds` keys on every join (the v7.6R join bug).
- **Residualization correctness** -- the background regression must use only the background set,
  never the test sets, or the null leaks signal.

## Done when
- The synapse/BBB run produces a real verdict on the rate-correlation leg (STRONG / WEAK /
  FALSIFIED), with cross-set covariation reported against within-set and background baselines.
- Saturated branches are filtered out of rate vectors before testing.
- Background normalization uses the randomized 300-gene set, not the 16-gene control.
- `tests/test_compute.py` covers residualization and the permutation null on a synthetic case
  with a known answer.
