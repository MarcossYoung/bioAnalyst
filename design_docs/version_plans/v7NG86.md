 Nullifier — Genomic dN/dS Keystone + Honesty Fixes

 Context

 The synapse/BBB run produced a confident NOVEL-UNTESTED verdict with a Genomic alignment 5.0/10 score while having run
 zero statistical tests — the genomic axis is computing on sand. Investigation (live Ensembl probes + code read)
 confirmed four interlocking defects, in priority order:

 1. Keystone — no trustworthy ω. tools/ensembl.py:333 _dnds() reads h["dn"]/h["ds"], but the current Ensembl REST
 homology endpoint returns a single combined dn_ds field. More importantly, Compara serves zero pairwise dN/dS in the
 current release — empirically dn_ds is None for 100% of orthologs across SYP/CAMK2A/SLC2A1/MFSD2A/BRCA2 (~90 each). So
 "pull Compara ω" recovers nothing. The only ω source today is the seqinr path off a separate fetch_gene_tree_aligned
 call, and the orthologs_missing_dn/ds diagnostic measures the (always-empty) Compara layer — so it reports "every
 ortholog missing" no matter what the estimator did. The fix is to compute ω in-process from the protein align_seq that
 the homology call already returns (present on 100% of orthologs), threading each ortholog's CDS onto it (pal2nal) and
 estimating ω with a pure-Python Nei–Gojobori (NG86) estimator (numpy is available; Biopython is not).
 2. dn_ds key bug — read dn_ds not dn/ds (low value alone since always null, but correct + stops mislabeling
 provenance).
 3. Saturation gate is global, not per-set — agents/analyst.py:511 _combine_saturation_flags OR-collapses Set A and Set
 B and only tests "near 1.0", so a degenerate Set A (ω≈0.042, ds-saturated) masqueraded as usable. Must flag each set
 independently (near-1.0 and degenerate-low/low-coverage) and refuse the cross-set comparison if either set is
 degraded.
 4. Honesty gate — agents/skeptic.py:187 forces genomic_evidence_alignment = 5 (and the prompt at line ~64 instructs
 the same) when untestable. Zero tests must yield N/A / excluded, never a neutral-looking 5.0.

 Intended outcome: real per-ortholog ω from the homology alignment with a correctness gate; honest per-set degradation
 flags; and a verdict that says "no genomic test run" with no fabricated genomic score.

 Verified facts (live API):
 - align_seq is protein, source & target mutually aligned (pairwise), same length.
 - ENSP → /lookup/id → Parent transcript → /sequence/id?type=cds returns codon-divisible nucleotide CDS (e.g.
 ENSMUSP00000069429 → ENSMUST00000069520 → 945 nt = 315 codons). Fetching CDS by the ENSP directly returns the peptide,
 not nucleotides — must resolve to the transcript.

 ---
 Fix 1 — Keystone: compute ω from homology align_seq (pal2nal + NG86)

 New estimator — tools/dnds.py (NEW, pure Python + numpy):
 - codon_align(prot_aln_a, prot_aln_b, cds_a, cds_b) -> (str, str) | None: pal2nal. Strip trailing stop codon; for each
 aligned protein column, emit the next codon from the corresponding CDS or --- for a gap. Correctness gate: translate
 each threaded CDS and assert it equals the ungapped align_seq; on mismatch return None (drop the ortholog — never
 fabricate ω).
 - ng86(codon_a, codon_b) -> {dnds, dn, ds, n_sites, s_sites}: Nei–Gojobori synonymous/nonsynonymous site &
 substitution counting with Jukes–Cantor correction. Return dnds=None when ds<=0, ds saturated (pS≥0.75 → JC
 undefined), or aligned length too short (configurable min codons, e.g. 30).

 tools/ensembl.py:
 - Parse align_seq, protein_id for source and each target into the ortholog dicts (extend get_orthologs /
 fetch_orthologs_by_id[_batch]). Fix _dnds() to read dn_ds (kept only as an opportunistic value; expected null).
 - Add resolve_cds_for_protein(protein_id) -> str | None: /lookup/id/{ENSP} → Parent transcript →
 fetch_cds_sequence(transcript, type=cds). Cache via the existing SQLite request cache. (Reuse fetch_cds_sequence,
 generalize it to accept a transcript id.)

 agents/analyst.py — replace the gene-tree+seqinr rate source:
 - Rewrite _fetch_rdnds_data to, per gene: take its one2one orthologs, batch-resolve source+target CDS (dedupe
 protein_ids, parallel like the existing ThreadPoolExecutor), then per ortholog run codon_align + ng86, yielding
 {species_lower: dnds}. This is a drop-in replacement for the seqinr output shape, so _attach_rdnds_to_orthologs
 (lowercase join at analyst.py:150) and the W3 per_gene_rate_vectors (genomic_data.py) consume it unchanged. Set
 dnds_source = "homology_pal2nal_ng86".
 - Keep r_bridge.pairwise_dnds available as an opt-in fallback (config flag) but no longer the default — avoids the
 R/seqinr run-device dependency and the per-gene Rscript spawn.
 - Fix the diagnostic (analyst.py ~437 dnds_diag): count orthologs_with_computed_dnds / dnds_source_counts over the
 computed ω, and rename/repoint orthologs_missing_dn/ds to "orthologs with no computable ω (no CDS / failed pal2nal /
 saturated)" so it reflects the real estimator, not the dead Compara field.

 Fix 2 — dn_ds key (folded into Fix 1)

 _dnds() reads homology.get("dn_ds"); ortholog parse drops the separate dn/ds keys (or maps both from dn_ds).
 Provenance dnds_source_counts no longer defaults bare orthologs to "ensembl_compara".

 Fix 3 — Per-set saturation / usability gate

 - In the per-set stats builder (agents/analyst.py _set_stats, ~482–498): keep dnds_saturation_flag (near-1.0 fraction
 > 0.5) and add dnds_degraded = saturated OR degenerate-low (e.g. median ω < floor with high ds-saturation share) OR
 dnds_n below a minimum-coverage threshold. Emit a per-set usable: bool + reason.
 - Replace _combine_saturation_flags with _set_usability(set_a_stats, set_b_stats) returning per-set flags plus
 cross_set_allowed = a.usable and b.usable and a reason naming the failing set(s). Keep a back-compat flag/reason for
 existing consumers.
 - compute layer (tools/compute.py): mirrortree_lite and _cross_set_analysis must check cross_set_allowed and return
 skipped=True / untestable with skip_reason naming which set is degraded — refusing the comparison rather than
 averaging through it.

 Fix 4 — Honesty gate (no score without a test)

 - agents/skeptic.py _apply_guardrails (185–190): determine genomic_test_ran = at least one primary genomic test
 actually executed (compute_results has a non-skipped test) and not untestable/degraded. When not ran: set
 scores["genomic_evidence_alignment"] = None (N/A), and note "No genomic test was run — genomic axis not scored." Do
 not emit 5.
 - Remove the "score 5 (neutral)" instruction from the skeptic system prompt (~line 64) and the _build_genomic_section
 fallbacks (~239, ~252) — replace with "report N/A; do not score."
 - Wherever a composite/overall uses the genomic score, exclude None from the average (drop from denominator).
 - Renderer report/renderer.py:128–130: skip/"N/A"-render scores whose value is None (current int(score) would crash).
 Frontend VerdictSection/VerdictCard + lib/types.ts: allow genomic_evidence_alignment: number | null and render "—".
