# Nullifier v7 — Concise Plan

Delta on v6. v6 is built. v7 makes outputs publication-grade.

1. **Branch-model ω via PAML codeml** — lineage-specific dN/dS replaces v6's pairwise Compara dN/dS for any lineage-specific claim.
2. **LOEUF integration** via gnomAD GraphQL — within-species human constraint complementing cross-species dN/dS.
3. **Compara metadata endpoints** — three new Ensembl endpoints used for provenance, method discovery, and failure diagnosis.
4. **Compute layer hardening** — sparse-results bug fixed; all tests return effect sizes + CIs + corrections by default.
5. **Phylostratigraphy consumed** from Liebeskind et al. 2016 (or equivalent published source) — no in-house dating.
6. **Tree visualization** — phylogenetic tree rendering with branch-model ω overlay.

R via rpy2 stays deferred. Closed-loop discovery (Generator agent) stays deferred to v8.

## 1. Backend changes

### Compute layer hardening (highest priority — fixes v6 bug)
v6 sometimes returns p-values without effect sizes or CIs. Fix:
- Every test wrapper returns a structured `TestResult` with **required** fields: `statistic`, `p_value`, `effect_size`, `effect_size_type`, `confidence_interval`, `correction_method`, `sample_sizes`, `assumptions_checked`. No optional fields.
- If a test can't produce one of these (e.g., Fisher's exact has no standard effect size), the field is explicitly `{"value": null, "reason": "..."}` rather than missing.
- Compute layer validates output schema before returning. Sparse results become impossible.
- Interpreter prompt updated to require referencing effect sizes and CIs, not just p-values.

### PAML codeml integration (new)
`tools/paml.py` — subprocess wrapper around PAML's codeml binary for branch-model ω.

Inputs the wrapper handles:
- Codon-aware alignment (fetch from Ensembl Compara's pre-computed alignments where available; PRANK fallback for the gaps)
- Newick tree with branches labeled for foreground/background designation
- Codeml control file generation (model = 2 for branch model, NSsites = 0, alternative + null hypothesis runs for LRT)

Outputs:
- ω per foreground branch
- Likelihood ratio test p-value (alternative vs. null model)
- Branch-specific dN, dS, dN/dS
- Provenance: PAML version, codeml control file hash, alignment source, tree source

Heavy operation — cache aggressively by `(gene_id, foreground_lineage, alignment_hash)`. TTL 90 days (PAML results don't change unless the input data changes).

Strict timeouts. PAML can hang on bad input — 5-minute wall clock per gene, return error rather than block the pipeline.

### LOEUF / gnomAD integration (new)
`tools/gnomad.py` — GraphQL client for gnomAD's API. Free, no auth.

Per gene: pull LOEUF, pLI, observed/expected ratios for missense and pLOF. Cache 30 days like Ensembl.

LOEUF feeds:
- Compute layer (constraint comparisons across gene sets)
- Provenance (record which gnomAD release was used)
- The Methodologist (LOEUF is now a tool the Methodologist can request for constraint analyses)

### Compara metadata endpoints (new)
`tools/ensembl.py` gains three thin wrappers:
- `list_compara_databases()` — `GET info/comparas`
- `list_compara_methods()` — `GET info/compara/methods`
- `get_method_species_sets(method)` — `GET info/compara/species_sets/:method`

Used for:
- **Provenance** — every Compara-derived result records the Compara release version
- **Method discovery** — before requesting orthologs for unusual species, check whether Compara has run protein trees for that species set
- **Failure diagnosis** — when homology returns empty, the diagnostic surfaces "Compara method X not run for species pair Y" instead of just "no results"

### Phylostratigraphy (consumed, not computed)
`tools/phylostratigraphy.py` — local lookup against a shipped dataset.

v7 ships with the Liebeskind et al. 2016 gene-age dataset as a static resource (CSV in the repo, version-stamped). Each human gene maps to an estimated phylostratum (cellular organism / eukaryote / opisthokont / metazoan / vertebrate / etc.) with the source paper's confidence rating.

The Methodologist can request "gene-set phylostratum distribution" as a tool. Compute layer aggregates and returns counts per stratum.

No in-house BLAST-based dating, no novel computation. The provenance records "phylostratigraphy from Liebeskind et al. 2016, retrieved version X."

If your father wants more recent phylostratigraphy data, the dataset file is swappable. The plumbing supports newer sources without code changes.

## 3. Frontend changes

### Tree visualization (new)
New component `PhylogenyView`. Renders a tree for a specific gene family with:
- Branch-model ω overlaid as branch color (heatmap: blue = constrained, red = accelerated)
- Duplication nodes annotated
- Foreground lineage(s) highlighted
- Branch ω values shown on hover, with the LRT p-value

Library: a minimal D3 tree renderer (no phylotree.js dependency — it's a heavy library for one chart). Server returns Newick + branch-ω JSON; client renders.

Renders when a run uses PAML branch-model analysis. Otherwise hidden.

### Compute results — effect sizes prominent
Update v6's compute results table to show effect size and CI alongside every p-value. p-values without effect sizes were the v6 bug; fixing the Compute layer fixes the display automatically.

Add a small "what does this effect size mean" tooltip per effect-size-type (Cliff's delta thresholds, Cohen's d thresholds).

### Provenance — Compara release surfaced
Update provenance affordances to show Compara release version, PAML version, gnomAD release when applicable. Already in v6 schema; just surface in the UI.

### Phylostratigraphy panel (new)
When a run includes phylostratigraphy analysis, render a small panel showing gene-set distribution across phylostrata, with the source dataset citation visible.

Everything else from v6 unchanged.

## 6. Open questions

1. **Foreground lineage selection.** PAML branch-model requires designating which branches are "foreground" (testing for accelerated evolution). Who decides? Options: (a) Methodologist proposes based on the hypothesis context, user confirms; (b) hardcoded common lineages (primates, rodents) selectable in the UI; (c) both. Recommend (c) — Methodologist proposes, UI lets user override before PAML runs.

2. **PAML timeout strategy.** 5 minutes per gene is generous but PAML can still hang or fail. When it does, what does the Interpreter say? Recommend: explicit "PAML branch-model analysis unavailable for [gene]; falling back to Compara pairwise dN/dS with explicit caveat in provenance." Don't silently substitute.

3. **Phylostratigraphy dataset version.** Liebeskind et al. 2016 is the most-cited reference but newer datasets exist (Casey et al. 2019, others). Use Liebeskind by default, document how to swap, or ship multiple and let the Methodologist pick? Recommend: ship Liebeskind only for v7, make the file swappable, defer multi-source support to v8.
