 Context

 The synaptic/BBB co-evolution hypothesis turns on correlated dN/dS rates between gene sets — yet every run reports
 dN/dS: n/a, the Spearman correlation tests come back blank, and the Skeptic flags "evolutionary rate correlations
 completely absent." The hypothesis is literally untestable in its current state.

 Root cause: the commit "added dN/dS calculations with R" shipped only scaffolding, never a working calculation. Both
 dN/dS sources are dead on the run device:

 1. Ensembl pairwise dn/ds — ensembl._dnds() (tools/ensembl.py:333-339) reads dn/ds from homology responses, but
 Compara returns these fields empty, so every orthologs[].dnds is None → _set_statistics yields dnds_mean = None →
 renders "n/a".
 2. PAML branch-model omega — paml.run_branch_model() shells out to the codeml binary, which isn't on PATH → status
 codeml_unavailable → no omega.

 tools/r_bridge.py is only a health check (initialize_r()/health_check()): it verifies seqinr/ape are installed but
 never calls them. The run device has R installed, and the required alignments are already being fetched
 (ensembl.fetch_gene_tree_aligned) — everything needed to compute dN/dS in R exists except the calculation itself.

 Outcome: implement a real seqinr::kaks() calculator (primary source, no codeml needed) covering all genes in the
 compared sets, so per-set dnds_mean populates and the Spearman "coordinated rates" tests actually run. Keep/document
 the codeml PAML path as a secondary source.

 Decisions (confirmed with user)

 - Source = Both: R seqinr::kaks as primary; codeml/PAML kept and documented.
 - Scope = all set genes: compute for every gene feeding the per-set stats and cross-set correlation tests (not just
 the ~8 starters), so Spearman gets ≥3 paired observations.

 Critical join detail (verified)

 Orthologs store target_species lowercase (pan_troglodytes, ensembl.py:314/357/497); gene-tree alignment keys are
 Title_Case (Pan_troglodytes, ensembl.py:544/634). The mapping in Step 3 must .lower() both sides or nothing matches.

 Implementation

 1. R calculator — backend/nullifier/tools/r_bridge.py

 Add pairwise_dnds(sequences: dict[str,str], reference="Homo_sapiens", use_cache=True) -> dict[str,float] | None:
 - Input is the sequences dict from fetch_gene_tree_aligned ({species_label: aligned_cds}).
 - Filter to seqs equal-length to reference, len % 3 == 0, reference present + ≥1 other. Else None.
 - SQLite cache mirroring tools/paml.py (_get_conn/_cache_get/_cache_set, ~/.nullifier/rdnds_cache.db), key =
 sha1(reference + sorted(seqs)).
 - Reuse R_HOME setup from initialize_r(). Import rpy2 inside a try/except Exception: return None (graceful — identical
 to today's "no dnds" state when rpy2/seqinr absent).
 - Build seqinr::as.alignment(...) → kaks(aln); ka/ks come back as dist → as.matrix(), index by name. Extract reference
 row only: dnds[sp] = ka[ref,sp]/ks[ref,sp], guarding ks<=0, NaN, Inf.
 - Return {sp.lower(): round(dnds,6)} (lowercased keys). Empty → None.
 - Risk: confirm the kaks return shape on the run device's seqinr version; wrap extraction defensively.

 2. Fetch pass — backend/nullifier/agents/analyst.py

 Add _fetch_rdnds_data(gene_data, genes, use_cache, on_event) modeled on _fetch_paml_data (analyst.py:53-90) but over
 all all_targets genes, not just starters:
 - Per gene: ensg = gene_data[g]["info"]["ensembl_id"] → ensembl.fetch_gene_tree_aligned(ensg) (Ensembl-cached, dedupes
 the PAML fetch) → r_bridge.pairwise_dnds(aligned["sequences"]).
 - Concurrency: parallelize the genetree fetch (ThreadPoolExecutor, respects the 14 req/s limiter), but run
 pairwise_dnds serially / behind a lock — rpy2's interpreter is not thread-safe.
 - Call in run_analyst (after _fetch_paml_data, before build_data, ~`analyst.py:158-164`).

 3. Map onto orthologs — analyst._attach_rdnds_to_orthologs(gene_data, rdnds_data)

 For each gene's orthologs, when o["dnds"] is None and o["target_species"].lower() is in the R result, set o["dnds"] =
 value and o["dnds_source"] = "r_seqinr_kaks". Writing into the existing dnds field means _set_statistics,
 genomic_data.per_gene_metrics/build_data pick it up with zero changes — including the variables["dnds"] vector
 Spearman consumes.

 4. Spearman tests now fire (no plumbing change)

 Because _genes_needing_dnds = full all_targets, ≥3 genes get a numeric per-gene dnds, so compute.spearman
 (tools/compute.py:357-387) no longer hits the <3 paired observations skip. dnds is already a METRICS/variable entry.
 Optional nudge: add one line to TEST_LIBRARY_DOC (compute.py:624-640) / methodologist.py noting per-gene pairwise dnds
 is available as a Spearman variable, so the LLM selects it for "coordinated rates."

 5. Provenance + wording (minimal)

 - interpreter.py:145 & system prompt :26 — drop the hardcoded "Ensembl never populates dn/ds"; numbers now appear when
 computed. Neutral empty-case string.
 - skeptic.py SKEPTIC_DNDS_LIMITATION — reword: pairwise dN/dS is R-computed when a Compara alignment exists; absence
 is coverage, not counter-evidence. Keep it injected.
 - Add a dnds_source count to _set_statistics dnds_diagnostics (ensembl vs r_seqinr_kaks).
 - Frontend: GenomicPanel.tsx (~66) add a (R seqinr)/(Ensembl) source tag + update the missing-note; extend
 AnalystSetStats in src/lib/types.ts. npm run build → backend/nullifier/static/.

 6. codeml path (secondary) — docs only

 Leave _fetch_paml_data starter-only and the graceful codeml_unavailable degradation. Add a README "Installing codeml
 (PAML)" section (put codeml on PATH; startup health confirms) and note pairwise dN/dS does not need codeml. Document R
 deps: pip install rpy2, install.packages(c("seqinr","ape")), [r] config in default_config.toml.

 7. Events (optional)

 Add analyst_rdnds_started/rdnds_gene_complete/analyst_rdnds_complete to events.py (mirror PAML events) for CLI/WS
 progress.
